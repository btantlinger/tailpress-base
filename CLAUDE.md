# TailPress Base Theme

A reusable WordPress theme framework with Tailwind CSS v4, Alpine.js, Vite, and ACF Flexible Content blocks.

## Architecture Overview

### CSS Pipeline
Entry point: `resources/css/app.css`
Import order matters — later imports override earlier ones:
1. `tailwindcss` — Tailwind v4 with @theme support
2. `../alpine-nav/css/nav-menu.css` — Navigation component styles (uses CSS variables from `variables.css`)
3. `./fonts.css` — Local @font-face declarations (if created for the project)
4. `./theme.css` — `@theme` block: brand colors, font families, breakpoints, text sizes
5. `./utilities.css` — WordPress alignment utilities (.alignfull, .alignwide)
6. `./custom.css` layer(utilities) — Project-specific styles (gradient effects, animations, etc.)
7. `./menu.css` — Navigation variable overrides (if created for the project)

**Key rule**: `theme.css` defines Tailwind design tokens via `@theme {}`. Colors defined there auto-generate utility classes (e.g. `--color-primary-500` → `bg-primary-500`, `text-primary-500`).

### Font Hosting (Self-Hosted for PageSpeed)
Fonts are always self-hosted, never loaded from Google Fonts CDN:
1. Download woff2 files to `resources/fonts/`
2. Create `resources/css/fonts.css` with `@font-face` declarations using `url('../fonts/filename.woff2')`
3. Import in `app.css` (after nav-menu.css, before theme.css)
4. Reference font names in `theme.css` `@theme` block (`--font-sans`, `--font-display`, `--font-mono`, etc.)
5. Vite handles font bundling — `vite.config.mjs` already has `assetsInclude` and `assetFileNames` for woff2/woff

### Vite Build System
- Dev: `npm run dev` (port 3000, HMR)
- Build: `npm run build` (outputs to `dist/`)
- **IMPORTANT**: When copying theme to a new project, update `base` path in `vite.config.mjs` from `/wp-content/themes/tailpress/dist/` to `/wp-content/themes/{new-name}/dist/`
- Entry points: `resources/css/app.css`, `resources/js/modules.js`, `resources/js/app.js`, `resources/css/editor-style.css`

### DDEV Integration
Required `web_extra_exposed_ports` in `.ddev/config.yaml`:
```yaml
web_extra_exposed_ports:
  - name: vite
    container_port: 5173
    http_port: 5172
    https_port: 5173
  - name: {project-name}
    container_port: 3000
    http_port: 3001
    https_port: 3000
```
The `get_vite_dev_server_url()` in `functions.php` builds `{scheme}://{host}:3000` for dev server detection. The ViteCompiler checks if the dev server is running and serves assets from it (dev) or from `dist/` manifest (production).

### ACF Block System
- **Define blocks**: Create PHP file in `includes/blocks/{name}.php` returning an array with `key`, `name`, `label`, `sub_fields`
- **Create template**: `template-parts/blocks/{name}.php` — uses `get_sub_field()` for data
- **Auto-loaded**: `content-builder.php` scans `includes/blocks/` directory automatically
- **Auto-injected fields**: Every block gets Spacing (padding/margin with responsive overrides) and Color (bg color + shade, text color + shade) tabs for free
- **Styling helper**: `wm_get_block_styling()` returns computed classes from the auto-injected fields
- **Page template**: `page-flexible.php` loops ACF flexible content and loads matching block templates

### Alpine Navigation System
Located in `resources/alpine-nav/`. Strategies: `drill-down`, `drop-down-panel`, `drop-down-fly-out-panel`, `content-panel`, `nested-list`.

- **variables.css**: ~270 CSS variables controlling all nav appearance. DO NOT edit this file per-project.
- **Per-project overrides**: Create `resources/css/menu.css` and import it last in `app.css`. Only override the variables you need to change.
- **nav-menu.php**: Template part with Alpine.js mobile drill-down menu. Loaded by `header.php`.
- **ACF fields on menu items**: Strategy selection, panel width, tile width, content blocks, descriptions — all configurable per top-level menu item.

### PHP Organization
- `functions.php` — TailPress Framework init, ViteCompiler, menu registration, script handling
- `includes/utils.php` — `wm_get_block_styling()`, responsive bg images, color darkness, markdown parsing
- `includes/content-builder.php` — Block auto-loading, field injection, ACF field group registration
- `includes/nav.php` — `Alpine_Nav_Walker`, `display_alpine_navigation()`, ACF nav fields
- `includes/theme-setup.php` — Image sizes, Gutenberg removal, hero preloading, template shortcode
- `includes/image-helpers.php` — Image rendering utilities

### Template Hierarchy
- `header.php` — HTML open, fixed header with logo + nav (via `template-parts/nav-menu.php`)
- `footer.php` — Footer with menus + copyright, HTML close
- `page-flexible.php` — ACF Flexible Content page builder loop
- `index.php` / `single.php` / `404.php` — Standard WordPress templates

## Per-Project Setup Checklist

When starting a new project from this base:

1. **Rename theme directory** and update:
   - `style.css` (Theme Name, Theme URI, Text Domain)
   - `vite.config.mjs` base path (`/wp-content/themes/{name}/dist/`)
   - `.ddev/config.yaml` web_extra_exposed_ports
2. **Brand colors**: Edit `resources/css/theme.css` `@theme` block — set `--color-primary`, `--color-secondary`, `--color-tertiary`. Either use `color-mix()` auto-palettes or hand-tune shade values.
3. **Fonts**: Download woff2 to `resources/fonts/`, create `resources/css/fonts.css`, import in `app.css`, update `--font-sans`/`--font-display`/`--font-mono` in `theme.css`.
4. **Navigation theme**: Create `resources/css/menu.css` to override Alpine Nav variables, import last in `app.css`.
5. **Header/Footer**: Customize `header.php` and `footer.php` for the project's design. Use `get_template_part('template-parts/logo', null, ['prefix' => 'nav', 'fill' => '#fff'])` for inline SVG logos.
6. **Build blocks**: Add PHP definitions in `includes/blocks/`, templates in `template-parts/blocks/`.

## Key Dependencies
- TailPress Framework v5.0.3 (Composer)
- Tailwind CSS v4 + @tailwindcss/vite (NPM)
- Alpine.js v3 + @alpinejs/anchor (NPM)
- Vite v6 (NPM)
- ACF Pro (WordPress plugin, required for blocks)

## Enqueue Handles
The ViteCompiler uses `tailpress` as the handle prefix. Script/style handles are:
- `tailpress-app` (CSS)
- `tailpress-modules` (JS)
- `vite-client` (dev only)

These handles are referenced in `functions.php` for footer placement and module type enforcement. Don't change them.
